## 3.5 Развёртывание и запуск прототипа

Для запуска прототипа tg-shop-ai требуется установленный Python версии 3.11 и менеджер пакетов pip. Проект содержит файл зависимостей `requirements.txt`, а также пример конфигурации `.env.example` для задания переменных окружения (в частности, токена Telegram-бота).

### Подготовка окружения
1) Клонировать репозиторий и перейти в каталог проекта.
2) Создать и активировать виртуальное окружение:
- Windows (PowerShell): `python -m venv venv` → `.\venv\Scripts\Activate.ps1`
3) Установить зависимости: `pip install -r requirements.txt`
4) Создать файл `.env` на основе `.env.example` и заполнить параметры (например, `BOT_TOKEN`).

### Инициализация базы данных и тестовых данных
Для подготовки базы данных и заполнения каталога тестовыми товарами используются режимы:
- `python main.py init-db`
- `python main.py seed-products`

### Запуск веб-интерфейса
Запуск web-части выполняется командой:
- `python main.py web`

После старта приложение доступно по адресу локального сервера (например, `http://127.0.0.1:5000/`) и предоставляет страницы: `/catalog`, `/admin/products`, `/orders`.

### Запуск Telegram-бота
Запуск Telegram-бота выполняется командой:
- `python main.py bot`

Для корректной работы требуется валидный токен бота в `.env` и доступ к сети.

Таким образом, проект поддерживает воспроизводимый сценарий развёртывания: подготовка окружения → инициализация БД → запуск web и bot, что позволяет быстро продемонстрировать работоспособность прототипа.
