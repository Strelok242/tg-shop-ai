Матрица требований ВКР/ПДП — tg-shop-ai
Репозиторий/код

 В корне есть main.py

 В корне есть requirements.txt

 README.md с инструкциями запуска

 .env не коммитится, есть .env.example

 Проверить к сдаче: ≥50 коммитов, ~2000 логических строк кода, сборка .exe (PyInstaller)

Архитектура/данные

 Клиент–сервер: Telegram Bot + Web + Backend + DB

 SQL БД используется (SQLite)

 Рекомендовано/опционально: PostgreSQL (решим отдельным шагом)

 Роли пользователей: минимум 3 (admin/manager/client)

 База данных: довести до ≥10 таблиц (по необходимости)

Telegram-бот (реализовано)

 /start — регистрация пользователя в БД

 /catalog — каталог товаров

 /buy <SKU> — создание заказа

 /myorders — список заказов

 /ai <текст> — AI-заглушка + логирование в ai_logs

 Довести к сдаче: ≥10 команд, ≥5 команд с параметрами, ≥5 нетекстовых ответов, диалог/FSM, меню настроек, выгрузка отчёта (xlsx/docx)

Web (реализовано)

 / — проверка работы

 /admin/products — добавление товара и список

 /catalog — публичный каталог

 /orders?tg_id=... — заказы пользователя (прототип)

 Довести к сдаче: ≥10 страниц, меню ≥10 пунктов, “хлебные крошки”, адаптивность/кроссбраузерность

Диплом/отчёты (предстоит)

 Полный текст ВКР по структуре кафедры + выводы по главам

 Список источников по ГОСТ

 Приложения: схемы (архитектура, ER), скриншоты, тесты
