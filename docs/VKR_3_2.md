## 3.2 Реализация веб-интерфейса (каталог, управление товарами, заказы)

Веб-часть проекта tg-shop-ai реализована в виде серверного приложения на Python (Flask) и предназначена для сценариев просмотра каталога, управления товарами и контроля заказов. Веб-интерфейс дополняет Telegram-бота и использует тот же слой доступа к данным и ту же базу данных, что обеспечивает единый источник данных по товарам и заказам.

### Точка входа и структура
Создание Flask-приложения выполнено по шаблону «фабрика приложения»: в модуле `app.web.app` определена функция `create_app()`, которая инициализирует приложение, подключает конфигурацию и подготавливает маршруты (обработчики HTTP-запросов). Отображение страниц реализовано через HTML-шаблоны (`templates`) и статические ресурсы (`static`).

### Основные маршруты и страницы
По результатам проверки маршрутов приложения реализованы следующие endpoints и URL:
- `index` — `/` (главная страница);
- `catalog_page` — `/catalog` (просмотр каталога товаров);
- `admin_products` — `/admin/products` (управление товарами; методы GET/POST);
- `orders_page` — `/orders` (просмотр заказов);
- `static` — `/static/<path:filename>` (выдача статических файлов).

Шаблоны страниц размещены в каталоге `app/web/templates` и включают: `catalog.html`, `admin_products.html`, `orders.html`.

### Работа с данными
Веб-часть взаимодействует с базой данных через ORM SQLAlchemy и общие модели проекта. Для отображения и управления каталогом используются данные таблицы `products`. Для контроля заказов используются таблицы `orders` и `order_items`. За счёт общей базы данных результаты действий пользователя в Telegram-боте (например, оформленные заказы) доступны для просмотра в веб-интерфейсе, что упрощает тестирование и демонстрацию интеграции компонентов.

Таким образом, веб-интерфейс обеспечивает административно-демонстрационный контур проекта и подтверждает корректность интеграции компонентов (бот → база данных → веб-отображение).
